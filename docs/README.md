## ğŸ¤ Contributing to Documentation

### Documentation Guidelines

1. **Keep it DRY** - Don't repeat information
2. **Link liberally** - Cross-reference related docs
3. **Use examples** - Show, don't just tell
4. **Update as you go** - Keep docs in sync with code
5. **Test your docs** - Ensure commands work

### Adding New Documentation

1. Determine which section it belongs to
2. Create the file in the appropriate folder
3. Add it to this index (docs/README.md)
4. Link from related documents
5. Update relevant checklists or guides

### Documentation Style

- Use **Markdown** for all documentation
- Include **code examples** that work
- Add **diagrams** for complex concepts
- Keep **line length** under 120 characters
